# Quality Bar

## Top Percentile UX

The app opens and the user wants to write.

Daymark is good enough only when:

- It feels useful with zero integrations.
- The first keystroke feels native.
- Slip is faster than switching to Apple Notes.
- Search feels instant enough to trust.
- Task completion feels tactile but quiet.
- The sidebar never creates anxiety.
- Suggestions feel grounded in visible context.

It is not good enough if it technically works but feels like a web app, has a default AI chat sidebar, has noisy badges, asks where to save every capture, or generates summaries nobody asked for.

## Top Percentile Performance

The user never notices saving. Typing, search, capture, and command palette interactions must feel local and immediate.

Any keystroke latency issue is a blocker.

## Top Percentile Design Craft

The app should look beautiful empty and calm when full.

Craft means:

- Native typography.
- Warm paper canvas.
- Translucent but quiet sidebar.
- Centered writing column.
- Cards that feel like documents, not chat bubbles.
- Motion that explains state and preserves spatial continuity.
- Hover states that are quiet but unmistakable.
- Color used sparingly and semantically.

Avoid SaaS dashboard chrome, generic glass, large gradients, colorful icon clutter, red badges, sparkle motifs, and over-rounded pill UI.

## Top Percentile Capability

Capability should deepen the writing loop:

- A messy thought can become a task.
- A task can become an open loop.
- An open loop can roll forward.
- A messy implementation thought can become a Codex spec.
- A meeting can be prepared from existing context.
- A person, project, or deal can accumulate memory over time.

Capability must not become feature sprawl.

## Top Percentile Reliability

Files never disappear. External edits are detected. Rollovers are idempotent. Search can be rebuilt. SQLite corruption does not destroy notes. Every generated task links back to source.
